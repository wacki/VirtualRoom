/*! \mainpage SLProject Welcome to the SLProject. SL stands for Scene Library and is a 3D computer graphics scenegraph library. It is developed at the Berne University of Applied Sciences. The main purpose of the project is to serve as starting framework for student and research projects. The framework is built in C++ and uses OpenGL ES 2.0 so that it can run also on mobile devices. Project files are provided for development environments on Windows, Linux, and Mac OSX. The demo application can also be built for Android (>=2.3.3) and Apple iOS (>=5). The framework can render alternatively with ray tracing and path tracing which provides in addition high quality transparencies, reflections and soft shadows. Please visit the project homepage at http://code.google.com/p/slproject/. The source code is provided under the GNU General Puplic License (GPL). The code is provided without any warranties whether expressed or implied.

Framework Structure:

  • The applications starting code depends on the operating system. In all cases we have the most outer shell of the application that handles the window and the OpenGL context creation and passes then the events to a C-function interface before it is handled by the C++-framework. The following OS are supported:
    • Windows, Linux & Max OSX applications can use the GLFW C-library for the platform independent window and context creation. GLFW is included in the SLProject repository.
    • Windows, Linux & Max OSX applications can use the Qt C++-library for the window and context creation. Qt has to be downloaded and installed additionally.
    • The Android application starts in JAVA and passes the events with JNI (Java Native Interface) to the C-interface.
    • On Apple iOS Devices the application starts in ObjectiveC before it passes the events to the C-interface.
    • You could in fact use any GUI library on any OS that can create OpenGL contexts. Other alternatives could be FLTK, Juce or wxWidgets.
  • The toplevel class of the framework is SLScene that represents the scene with its properties. The scene content is created in SLScene::onLoad.
  • SLSceneView represents a dynamic real time 3D view onto the scene. A scene can be shown in multiple sceneviews as demonstrated in the app-Viewer-Qt application. An sceneview receives all events (keyboard, mouse, OpenGL etc.) from the GUI via the C-Interface (see slInterface).
  • The scenegraph hierarchy is built with the class SLNode that can have 0-N children nodes and 0-N triangle meshes. A node can be transformed (translated, rotated and scaled) in 3D-space.
  • The SLMesh class is the base class for triangulated or wireframed meshes. A mesh is rendered with a material defined in SLMaterial.
  • The classes SLRevolver, SLSphere, SLCylinder, SLCone, SLBox, SLPolygon and SLRectangle are all inheritants from SLMesh.
  • The space partitioning is implemented in the classes SLAABBox and SLUniformGrid.
  • All OpenGL ES 2.0 code is restricted to the classes SLGLState, SLGLTexture, SLGLBuffer, SLGLShader and SLGLShaderProg.
  • The lighting is implemented in the classes SLMaterial, SLLight, SLLightSphere and SLLightRect.
  • The classes for the 2D-GUI are SLTexFont, SLText and SLButton and can be used for a simple GUI.
  • The ray tracer is implemented in the classes SLRaytracer and SLRay.
  • The linear algebra math is implemented in the classes SLMat3, SLMat4, SLVec3, SLVec4 and SLQuat.

Framework Dependencies:

  • The framework uses beside OpenGL the following exteral libraries that are also included in the SLProject repository:
    • JPEG for JPEG image type loading.
    • PNG image type loading.
    • ZLIB for compression of PNG images.
    • nvwa for memory management.
    • assimp for 3D file formats loading.
    • libovr for Oculus Rift control.
